# Скиллы и кастомные команды OMP (Oh My Pi)

Справка по всем скиллам и кастомным (плагинным) слэш-командам, доступным в этой рабочей среде OMP. Составлено по фактическому составу подключённых плагинов: `superpowers`, `vercel`, `caveman`.

## Скиллы

Скилл — это специализированная инструкция, которую агент обязан прочитать (`skill://<name>`) перед началом соответствующей работы, если задача попадает под её описание.

### Vercel-платформа

| Скилл | Описание |
|---|---|
| `ai-gateway` | Маршрутизация моделей через Vercel AI Gateway: отказоустойчивость провайдеров, учёт стоимости, единый API для нескольких AI-провайдеров. |
| `ai-sdk` | Vercel AI SDK: чат-интерфейсы, генерация текста, структурированный вывод, вызов инструментов (tool calling), агенты, интеграция MCP, стриминг, эмбеддинги, реранкинг, генерация изображений — под любые LLM-провайдеры. |
| `auth` | Интеграция аутентификации в Next.js — Clerk, Descope, Auth0: middleware-паттерны, формы входа/регистрации, provisioning через Vercel Marketplace. |
| `bootstrap` | Первичная настройка репозитория, завязанного на ресурсы Vercel (БД, auth, интеграции): правильный порядок линковки, provisioning окружения, `env pull`, первый запуск db/dev. |
| `chat-sdk` | Vercel Chat SDK: мультиплатформенные боты (Slack, Telegram, Teams, Discord, Google Chat, GitHub, Linear) на одной кодовой базе. |
| `deployments-cicd` | Деплой и CI/CD на Vercel: promote, rollback, инспекция деплоев, сборка с `--prebuilt`, конфигурация CI-воркфлоу. |
| `env-vars` | Переменные окружения Vercel: `.env`-файлы, команды `vercel env`, OIDC-токены, конфигурация под конкретные окружения. |
| `knowledge-update` | Исправляет устаревшие знания модели о платформе Vercel, вводит новые продукты — подключается в начале сессии автоматически. |
| `marketplace` | Vercel Marketplace: поиск и установка интеграций, авто-provisioning env-переменных, единый биллинг, CLI `vercel integration`. |
| `microfrontends` | Построение и деплой микрофронтендов на Vercel: multi-zones, независимые деплои по командам, кросс-роутинг. |
| `next-cache-components` | Next.js 16 Cache Components: PPR, директива `use cache`, `cacheLife`, `cacheTag`, `updateTag`, миграция с `unstable_cache`. |
| `next-forge` | Стартер `next-forge` — продакшн-грейд Turborepo-монорепо для SaaS от Vercel. |
| `next-upgrade` | Обновление Next.js по официальным гайдам миграции и кодмодам. |
| `nextjs` | App Router целиком: роутинг, Server Components, Server Actions, Cache Components, layouts, middleware, data fetching, деплой на Vercel. |
| `routing-middleware` | Vercel Routing Middleware: перехват запроса до кэша, rewrites, redirects, персонализация (Edge/Node/Bun runtime). |
| `runtime-cache` | Vercel Runtime Cache API: эфемерный per-region key-value кэш с инвалидацией по тегам. |
| `shadcn` | shadcn/ui: CLI, установка компонентов, паттерны композиции, кастомные реестры, темизация, Tailwind. |
| `turbopack` | Настройка бандлера Next.js (Turbopack): HMR, отладка сборки, отличия от Webpack. |
| `vercel-agent` | Vercel Agent: AI-ревью PR, расследование инцидентов, установка SDK. |
| `vercel-cli` | Vercel CLI: деплой, env-переменные, линковка проектов, логи, метрики, домены. |
| `vercel-firewall` | Vercel Firewall: авто-защита от DDoS, WAF (кастомные правила, блокировка IP, rate limiting), Attack Mode. |
| `vercel-functions` | Vercel Functions: Serverless/Edge Functions, Fluid Compute, стриминг, Cron Jobs, конфигурация рантайма. |
| `vercel-sandbox` | Vercel Sandbox: эфемерные Firecracker microVM для безопасного запуска недоверенного/сгенерированного кода. |
| `vercel-storage` | Выбор и настройка хранилища: Blob, Edge Config, Marketplace storage (Neon Postgres, Upstash Redis). |
| `workflow` | Vercel Workflow DevKit: устойчивые долгоживущие воркфлоу с паузой/резюме, ретраями, шаговым выполнением. |
| `react-best-practices` | Чек-лист лучших практик React для TSX-файлов: структура компонентов, хуки, доступность, производительность, типизация. |

### Процесс разработки (superpowers)

| Скилл | Описание |
|---|---|
| `using-superpowers` | ОБЯЗАТЕЛЕН в начале любого разговора — объясняет, как находить и подключать остальные скиллы. |
| `brainstorming` | ОБЯЗАТЕЛЕН перед любой творческой работой (новая фича/компонент/поведение) — выясняет намерение, требования и дизайн до реализации. |
| `writing-plans` | Превращает спецификацию/требования в пошаговый план реализации до написания кода. |
| `executing-plans` | Выполнение готового плана реализации в отдельной сессии с контрольными точками ревью. |
| `subagent-driven-development` | Выполнение плана с независимыми задачами через субагентов в текущей сессии. |
| `dispatching-parallel-agents` | Когда есть 2+ независимые задачи без общего состояния — параллельный запуск через субагентов. |
| `using-git-worktrees` | Изоляция рабочего пространства перед началом фичи или выполнением плана через git worktree. |
| `test-driven-development` | Обязателен при реализации любой фичи/багфикса — тесты пишутся до кода реализации. |
| `systematic-debugging` | ОБЯЗАТЕЛЕН при любом баге, падении теста или неожиданном поведении — поиск первопричины до предложения фикса. |
| `verification-before-completion` | Перед тем как заявить «готово / исправлено / проходит» — обязательный прогон команд верификации, доказательства раньше утверждений. |
| `verification` | Сквозная проверка полного флоу (браузер → API → данные → ответ); триггерится при старте dev-сервера или вопросе «почему не работает». |
| `receiving-code-review` | Как принимать фидбек по код-ревью: техническая проверка и верификация вместо автоматического согласия. |
| `requesting-code-review` | Запрос ревью при завершении задач/фич — перед мержем. |
| `finishing-a-development-branch` | Когда реализация готова и тесты проходят — структурированные варианты завершения (merge/PR/cleanup). |
| `writing-skills` | Создание, редактирование и проверка скиллов перед деплоем. |

### Прочее

| Скилл | Описание |
|---|---|
| `caveman` | Режим ультра-сжатого общения (экономия токенов ~75%) с сохранением технической точности. Уровни: `lite`, `full` (по умолчанию), `ultra`. Триггерится фразами «caveman mode», «be brief» и т. п. |
| `educational-video-and-text-summary` | Плотный конспект обучающих видео/статей/лекций без «воды»: ключевые моменты, концепции, шаги, примеры, предупреждения. |
| `last30days` | Исследование темы за последние 30 дней по соцсетям (Reddit, X, YouTube, TikTok, Instagram, HN, Polymarket) — экспертиза + готовые промпты. |

## Кастомные команды

Файловые слэш-команды из установленных плагинов (`~/.claude/plugins/cache/...`). Доступны только команды плагина `vercel` — у `superpowers` и `caveman` собственных командных файлов нет, они работают исключительно через скиллы.

| Команда | Описание |
|---|---|
| `/vercel:bootstrap` | Детерминированный флоу первичной настройки репозитория: preflight-проверки, provisioning интеграций, проверка env-ключей, безопасный запуск db/dev. |
| `/vercel:env` | Управление переменными окружения Vercel — `list`, `pull`, `add`, `remove`, `diff`. Значения переменных никогда не выводятся в текст ответа (только имена/окружения/метаданные). |
| `/vercel:deploy` | Деплой текущего проекта на Vercel через CLI с preflight-проверками и явным подтверждением для продакшена. По умолчанию — preview-деплой; аргумент `prod`/`production` — деплой в продакшен. |
| `/vercel:marketplace` | Поиск, установка и применение интеграций Vercel Marketplace (БД, CMS, auth-провайдеры и т. п.) с пошаговой настройкой и локальной проверкой. |
| `/vercel:status` | Комплексная диагностика проекта: последние деплои, привязка проекта, домены, статус сборки, состояние окружения. |

*Примечание:* название команды образуется как `<плагин>:<команда>` — при отсутствии конфликта имён её можно вызывать и коротким алиасом (например `/deploy` вместо `/vercel:deploy`), если имя не занято другим источником с более высоким приоритетом.
